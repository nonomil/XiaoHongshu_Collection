# UI 简化与设置弹窗改版设计

日期：2026-03-13

## 背景
当前 UI 将配置面板常驻在主界面右侧，视觉负担大、操作区被压缩，用户反馈“界面太复杂”。需求是：主界面保留简短摘要区 + 右上角“设置”按钮弹出配置弹窗，其余布局聚焦输入与日志。

## 目标
- 主界面更简洁，用户第一眼聚焦于“输入与执行”。
- 设置项仍完整保留，但默认隐藏在弹窗中。
- 批量执行时提供每条链接的进度反馈，避免“卡死感”。
- 维持现有 API 能力与后端逻辑，减少破坏性改动。

## 非目标
- 不更换后端保存与导出逻辑。
- 不引入外部前端框架。
- 不调整保存/导出核心流程的业务规则。

## 方案对比与选择
1. 设置弹窗（推荐）
- 优点：最强收敛，视觉最清爽，操作区最大化。
- 代价：设置需要额外点击。

2. 侧边抽屉
- 优点：仍可同时观察日志。
- 代价：页面宽度被占用，复杂感回归。

3. 主界面折叠设置区
- 优点：实现成本低。
- 代价：主界面仍显拥挤。

选择：设置弹窗。

## 信息架构与布局
主界面分为三块：
1) 顶部栏
- 左侧标题与简短说明
- 右侧“设置”按钮

2) 摘要区（紧凑一行）
- 输出路径（短显示，hover 显完整）
- 冲突策略（标签）
- AI/OCR 开关（标签）
- 每条图片上限

3) 操作区 + 日志区
- 操作区：
  - 链接保存卡片（输入框 + 保存/清空）
  - 收藏导出卡片（说明 + 按钮）
- 日志区：
  - 总览卡（总数/成功/失败）
  - 逐条进度列表（等待/运行中/完成/失败）
  - 原始 JSON 折叠区

## 设置弹窗内容
分区结构：
- 路径设置
- 命名规则
- 运行参数（AI/OCR/超时/上限）
- 展示开关（显示原始 JSON）

交互：
- 点击右上角“设置”打开
- 点击遮罩或 ESC 关闭
- 保存后关闭弹窗并刷新摘要区

## 交互与数据流
- 保存链接：
  - 前端调用 `/api/save-links-stream`，接收 NDJSON 事件
  - 每条链接更新进度条状态
- 收藏导出：
  - 保留原同步调用与结果展示

## 错误处理
- 单条失败：进度条标红，并在 tooltip 展示错误
- 批次失败：在状态区提示错误，原始 JSON 保留

## 可访问性
- 弹窗支持 ESC 关闭
- 按钮与输入可键盘访问
- 颜色区分状态同时保留文案

## 测试与验证
- `npm test` 全量通过
- 手动验证：
  - 设置弹窗打开/关闭
  - 摘要区更新
  - 多条链接逐条进度
  - 原始 JSON 折叠展示

## 变更影响
- UI 结构调整：移除常驻设置面板，新增弹窗层
- 保持 API 与业务逻辑不变
