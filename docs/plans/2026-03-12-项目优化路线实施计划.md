# 项目优化路线 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 把当前仓库收敛成稳定的小红书归档引擎，统一入口、配置、错误处理和输出结构，为后续自用、分享和产品化打基础。

**Architecture:** 保留现有抓取、OCR、评论和 Markdown 输出能力，但先抽象统一任务管线和配置模型，再让 CLI、本地 UI 和批处理入口复用这套内核。实现上优先保护现有可用性，通过测试和渐进替换避免一次性重构带来的回归。

**Tech Stack:** Node.js、现有 CDP 脚本、现有 `node --test` 测试体系、静态 UI、JSON 配置。
