# 外部入口卡片（通用收件箱）设计

**日期：** 2026-03-15

## 背景
当前主页面只有“链接保存”“收藏导出”两个入口，Pushbullet 收件箱同步存在但容易被忽略。后续还可能接入 IFTTT、飞书等外部来源，需要一个更通用的入口承载。

## 目标
- 在主页面新增“入口 03”卡片，名称与文案保持通用，不绑定具体供应商。
- 仍复用现有后端 `/api/inbox/sync` 行为，不引入新的数据结构。
- 让用户一眼理解：这是外部来源的入口，当前已接入 Pushbullet，未来可扩展。

## 非目标
- 不引入新的 provider 选择器或多供应商配置。
- 不改动后端同步逻辑、不新增 API。

## 方案概述
- 主页面新增卡片：`入口 03：外部入口`。
- 卡片描述：强调“外部服务同步到本地收件箱”，并提示当前接入 Pushbullet、后续可扩展。
- 操作按钮：`同步外部收件箱`，触发既有 `/api/inbox/sync`。
- 设置面板标题改为“外部入口设置”，但字段仍明确标注 Pushbullet（保持可理解性）。

## UI 文案与结构
- 卡片标题：外部入口
- 卡片说明：从外部服务收集链接并同步到本地收件箱。当前接入 Pushbullet，后续可扩展 IFTTT、飞书等。
- 卡片按钮：同步外部收件箱
- 设置面板分组标题：外部入口设置

## 数据流
- UI 点击卡片按钮 -> `/api/inbox/sync`
- 读取 UI 配置（Pushbullet token、inbox path） -> 后端同步
- 返回报告 -> UI 渲染结果

## 错误与提示
- 沿用已有错误展示逻辑，不新增错误类型。

## 测试
- 复用现有 UI/服务器测试（`scripts/ai/__tests__/ui_server.test.js`）。
- 不新增新的单元测试，仅校验现有测试通过。
