# 小红书做深 Detailed Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 把当前仓库打磨成一个稳定、清晰、可长期维护的小红书归档引擎，优先统一任务管线、配置、错误处理、输出结构和评论提炼能力。

**Architecture:** 保留现有收藏夹抓取、单笔记保存、OCR、评论归档、AI 摘要和本地 UI 能力，但不再继续横向扩平台，而是把这些能力收敛到同一条小红书专用 pipeline：`input -> fetch -> enrich -> write -> report`。实现时先用测试固定现状，再分步抽出共享模型、共享配置、共享错误和共享报告，最后再增强评论质量和 UI 入口一致性。

**Tech Stack:** Node.js、现有 Chrome CDP 脚本、现有 `node --test` 测试体系、静态 UI、JSON 配置、`tesseract.js`、OpenRouter。
