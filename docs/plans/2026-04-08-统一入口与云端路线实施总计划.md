# 统一入口与云端路线实施总计划 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 为当前项目建立一条可执行的总路线，先修复 Pushbullet 历史导入与本地归档，再逐步建设统一 ingress、本地/云端分工、外部入口接入与同步模型。

**Architecture:** 保持当前仓库继续担任“执行器内核”，不把小红书抓取逻辑迁出。新增统一入口协议与队列层，把 Pushbullet、浏览器插件、飞书、OpenClaw、未来 IMA CLI 统一收敛为入口；云端优先承载 ingress、队列和适合后处理的来源，本地继续承载小红书强登录态执行链路。

**Tech Stack:** Node.js、现有 `scripts/` + `ui/`、JSONL inbox、未来 ingress API、Chrome/CDP、本地 UI、Pushbullet API、飞书 webhook、OpenClaw/ClawCloud、可选 SQLite
