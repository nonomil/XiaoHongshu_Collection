# 2026-03-24 设置弹层页签化设计

## 背景

当前首页主流程已经收口到三大入口，但设置弹层仍然是单页长表单：

- 桌面端滚动距离长，保存按钮容易被埋到底部
- 移动端进入设置后，用户要反复滚动才能找到浏览器或收件箱配置
- 字段很多，但常用项和低频项没有层级

本轮目标不是重做配置模型，而是在不改变现有 `ui-config` 结构和字段 `id` 的前提下，把设置区做成更可扫描、更容易保存的结构。

## 方案比较

### 方案 A：继续单页长表单，只做间距优化

- 优点：改动最小
- 缺点：核心问题不变，用户仍然要在长表单里找字段和找保存按钮

### 方案 B：折叠面板

- 优点：比长表单更短
- 缺点：展开收起状态多，移动端来回折叠仍然费操作

### 方案 C：页签分组 + 底部固定操作区

- 优点：路径清晰，常用项优先，保存按钮始终可见
- 缺点：需要补一点交互逻辑和测试

## 选型

采用方案 C。

分组固定为四类：

1. 基础保存：路径、命名
2. 浏览器接入：Chrome 接管、无头、登录浏览器
3. 外部入口：Pushbullet、收件箱归档
4. 高级参数：AI、OCR、超时、展示

## 设计约束

- 不修改现有字段 `id`
- 不改变 `readConfigFromForm()` / `applyConfigToForm()` 的数据结构
- 设置仍由同一个 `<form id="config-form">` 提交
- 需要保留键盘与屏幕阅读器可识别的 `tablist/tab/tabpanel`
- 移动端页签要能折成更紧凑的网格

## 验收点

- 设置弹层默认进入 `基础保存`
- 点击页签只切换面板，不影响已有配置读写
- 关闭后再打开，当前页签仍保持
- `保存设置` 与 `重新加载` 在弹层底部持续可见
- 相关 UI 测试与全量测试通过
