# 2026-03-12 小红书 UI 升级设计

## 目标

- UI 支持“路径配置、命名规则、运行参数、结果展示”的一站式配置。
- UI 配置持久化到 `config/ui.json`，仅在 UI 运行时覆盖，不影响 CLI 默认行为。
- 输出文件冲突按“正文一致覆盖、正文不同追加后缀”的规则处理。
- 结果展示更直观（摘要 + 文件清单 + 原始 report）。

## 非目标

- 不引入多平台能力。
- 不改变 CLI 默认输出路径与命名规则。
- 不新增远程依赖或外部服务。

## UI 设计要点

- 路径区：
  - 链接保存：`outputRoot`、`imagesRoot`
  - 收藏导出：`collectionOutputRoot`、`collectionRawPath`
- 命名/冲突区：
  - 默认 `标题.md`
  - 冲突处理：读取已有 Markdown，剔除 YAML 头，仅比较正文
  - 正文相同 => 覆盖；正文不同 => 追加 `-1/-2`
- 运行参数区：
  - AI 总结开关
  - Vision OCR 开关
  - OCR fallback 开关
  - 超时/最大图片数
- 结果区：
  - 摘要统计 + 文件清单 + 输出路径
  - 原始 report JSON 折叠显示

## 配置与接口

- 配置文件：`config/ui.json`
- UI 加载/保存：前端读取/提交；后端合并默认值 + 用户值
- 运行时覆盖参数不写回 `openrouter.json` / `vision-ocr.json`

建议的 `ui.json` 结构（示意）：

```json
{
  "paths": {
    "saveLinksOutputRoot": "<path>",
    "saveLinksImagesRoot": "<path>",
    "collectionOutputRoot": "<path>",
    "collectionRawPath": "<path>"
  },
  "naming": {
    "conflictStrategy": "content-aware",
    "maxTitleLength": 80
  },
  "runtime": {
    "aiSummaryEnabled": true,
    "visionOcrEnabled": true,
    "ocrFallbackEnabled": true,
    "openRouterTimeoutMs": 30000,
    "visionOcrTimeoutMs": 60000,
    "maxImagesPerNote": 12
  },
  "ui": {
    "showRawReport": true
  }
}
```

## 数据流

1. UI 读取 `config/ui.json`，渲染表单。
2. 提交时携带 `uiConfig`，服务端与默认配置合并。
3. 后端在调用保存流程时注入覆盖参数：
   - 链接保存：覆盖 `outputRoot` / `imagesRoot` / 命名规则
   - 收藏导出：覆盖 `rawPath` / `outputRoot` / 命名规则
4. 保存流程按冲突策略决定覆盖或追加后缀。

## 命名冲突算法

- 读取目标 Markdown（若存在）
- 剔除 YAML 头部，仅比较正文
- 正文一致 => 直接覆盖（写入新 frontmatter）
- 正文不同 => 追加 `-1/-2` 直到无冲突或正文一致

## 测试计划

- 正文比较与命名冲突策略（覆盖 vs 追加）
- `ui_server` 接收覆盖参数并透传
- UI 配置读写与默认值合并

## 兼容性与风险

- UI 覆盖仅作用于 UI 请求，不影响 CLI 运行。
- 若 `ui.json` 缺失或无效，全部回退默认行为。
