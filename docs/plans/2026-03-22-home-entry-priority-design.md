# 首页入口优先级重排设计

Date: 2026-03-22

## 背景

首页当前已经具备三条主路径：

- 入口 01：手动粘贴链接保存
- 入口 02：收藏导出 + OCR
- 入口 03：外部收件箱同步 + 解析保存

最近的真实使用反馈说明，入口 02 和入口 03 属于“登录浏览器后就想立刻点击执行”的高频批量入口，但它们原本位于手动粘贴卡片之后，视觉优先级不够高，也不够像一组相关动作。

## 目标

- 让入口 02 和入口 03 在桌面端明确处于同一排
- 把这组批量入口前置，减少滚动和来回寻找
- 增大关键按钮点击区域，让“开始导出收藏”和“收件箱解析保存”更容易命中
- 保持移动端单列，不引入新的交互复杂度

## 非目标

- 不改动后端 API
- 不改动入口 01 的保存逻辑
- 不新增新的首页入口
- 不重做整体视觉语言，继续沿用现有暖色纸面风格

## 设计决策

### 1. 把高频批量入口前置成一个独立分组

新增 `entry-secondary-shell`，把入口 02 和入口 03 放入同一个“快捷批量入口”区块。

原因：

- 这两项都依赖“浏览器已登录后直接执行”的心智模型
- 用户点击时更像是在“选择批处理方式”，而不是在三个完全独立的卡片之间跳转
- 分组后更适合后续继续扩展公众号 / 知乎 / CSDN 的批量来源入口

### 2. 入口 01 下沉为手动补充入口

入口 01 仍保留完整的大文本输入，但不再占据首页首屏最高优先级。

原因：

- 它是通用入口，但不是最近最高频的入口
- 下沉后，首页首屏更贴合“先跑批量流程，再做人工补充”的真实使用顺序

### 3. 批量卡片按钮改为全宽

为入口 02 的主按钮和入口 03 的“收件箱解析保存”按钮增加全宽样式。

原因：

- 首页上这两个动作都属于明确的“主 CTA”
- 全宽按钮在桌面端更醒目，在移动端也更容易点击

### 4. 保持现有响应式断点

桌面端继续使用 `0.9fr / 1.1fr` 双列，移动端仍在 `980px` 以下切回单列。

原因：

- 入口 03 的步骤结构天然比入口 02 更长，需要略宽的列宽
- 保持现有断点，能减少回归风险

## 已落地内容

- 新增“快捷批量入口”头部说明
- 入口 02 和入口 03 前置并保持同排
- 入口 02 增加更明确的场景文案
- 入口 03 保留 Step 1 / Step 2 结构，但把主动作做得更易点击
- 入口 01 仍完整保留，只是下移为手动入口

## 后续建议

如果继续优化首页，建议只做两类小步迭代：

1. 在快捷批量入口头部增加“最近成功时间 / 最近同步条数”这种轻量运行摘要。
2. 为入口 02 和入口 03 增加更明确的空闲 / 运行中状态提示，而不是再做结构级大改。
