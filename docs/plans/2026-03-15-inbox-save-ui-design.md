# 收件箱同步入口增强与解析保存 Design

**目标**
- 顶栏增加“同步全部”按钮，避免只看得到“同步最新”。
- 外部入口卡片增加“收件箱解析保存”按钮，用于把收件箱链接解析并保存为笔记。
- 保持“同步”和“解析保存”分离，不自动串联。
- 解析保存后不清理收件箱内容（保留所有链接）。

**非目标**
- 不实现自动同步后自动解析保存。
- 不删除或清空收件箱数据。
- 不改动现有 Pushbullet 同步逻辑与鉴权方式。

**方案概述**
- UI：
  - 顶栏新增按钮“同步全部”。
  - 外部入口卡片按钮区新增“收件箱解析保存”。
- API：新增 `POST /api/inbox/save`，调用 `saveInboxUrls` 执行解析保存。
- 结果展示：复用现有结果区与摘要渲染逻辑。

**UI 交互与文案**
- 顶栏按钮：`同步最新` + `同步全部`。
- 外部入口卡片按钮区：`同步最新` / `同步全部` / `收件箱解析保存`。
- 解析保存状态文案：
  - 开始：`正在解析保存收件箱...`
  - 成功：`收件箱解析保存完成`
  - 失败：`收件箱解析保存失败`

**后端行为**
- `/api/inbox/save`：
  - 调用 `saveInboxUrls` 读取收件箱文件并调用保存流程。
  - 不清理收件箱内容。
  - 返回结构化结果（total, successCount, failureCount, results）。

**错误处理**
- API 失败返回统一错误结构，UI 通过现有错误栏展示。
- 收件箱为空时：返回总数为 0，并在 UI 中显示结果摘要。

**测试策略**
- `ui_index.test.js`：断言新增按钮文案存在（顶栏“同步全部”、卡片“收件箱解析保存”）。
- `ui_server.test.js`：新增 `/api/inbox/save` 的成功返回测试。
- 如需要：为 `saveInboxUrls` 的 UI 透传参数补充单测。
